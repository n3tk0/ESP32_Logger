## 2. NEW Findings — Core / Logger.ino

### N1: Division by Zero in Test Mode LED Blink
- **Severity:** CRITICAL
- **File:** `Logger.ino:520`
- **Code:** `digitalWrite(pin, (millis() / config.flowMeter.blinkDuration) % 2);`
- **Issue:** If `config.flowMeter.blinkDuration == 0`, modulo-zero crashes the device.
- **Fix:** Guard with `if (config.flowMeter.blinkDuration > 0) { ... }`

### N2: Format String Mismatch — earlyGPIO_bitmask
- **Severity:** HIGH
- **File:** `Logger.ino:622`
- **Code:** `snprintf(line, sizeof(line), "...bitmask:0x%04X...", earlyGPIO_bitmask);`
- **Issue:** `earlyGPIO_bitmask` is `uint32_t` but format is `%04X` (16-bit). Should be `%08lX`.
- **Impact:** Truncates upper 16 bits of GPIO state in log output.

### N3: Bit Shift Out of Bounds on User-Configurable Pin
- **Severity:** HIGH
- **File:** `Logger.ino:617-619`
- **Code:** `(earlyGPIO_bitmask >> config.hardware.pinWakeupFF) & 1`
- **Issue:** If user configures pin > 31, right-shift on `uint32_t` is undefined behavior.
- **Fix:** Add `if (pin < 32)` guard before each shift.

### N4: `cycleTotalPulses` Not Volatile
- **Severity:** MEDIUM
- **File:** `Logger.ino:637`
- **Code:** `noInterrupts(); cycleTotalPulses += pulseCount; pulseCount = 0; interrupts();`
- **Issue:** `cycleTotalPulses` is NOT declared volatile but is modified inside `noInterrupts()` block alongside ISR-sourced `pulseCount`. Compiler may cache the value across the critical section.
- **Fix:** Declare `volatile uint32_t cycleTotalPulses`.

### N5: `strncpy` Without Explicit Null Termination
- **Severity:** MEDIUM
- **File:** `SensorTypes.h:44-48`
- **Code:** `strncpy(r.sensorId, id, sizeof(r.sensorId) - 1);`
- **Issue:** Constructor does `memset(&r, 0, ...)` first (safe), but `strncpy` itself does not guarantee null-termination if source >= dest size. The `memset` pre-zeroing makes this safe *in practice*, but the pattern is fragile — any future refactor removing `memset` breaks it.
- **Recommendation:** Add explicit `r.sensorId[sizeof(r.sensorId)-1] = '\0';` after each `strncpy`.

### N6: Version String Mismatch
- **Severity:** LOW
- **File:** `Logger.ino:1-2` vs `full_logger.ino:2`
- **Issue:** `Logger.ino` reports v4.2.0, `full_logger.ino` reports v4.1.4. Inconsistent version across entry points.

## 3. NEW Findings — Pipeline & Tasks

### N7: AggregationEngine `bucket()` Off-by-One
- **Severity:** HIGH
- **File:** `AggregationEngine.cpp:158`
- **Code:** `while (i <= inLen && outCount < outMaxLen)`
- **Issue:** Loop allows `i == inLen`, then accesses `in[i].timestamp` on next iteration. Short-circuit via `endOfData = (i >= inLen)` at line 159 prevents the access in most paths, but the loop condition itself is wrong — `i <= inLen` should be `i < inLen` for safety.
- **Risk:** Buffer over-read if short-circuit logic is ever refactored.
- **Fix:** Change to `i < inLen`.

### N8: ProcessingTask — No Drop Counter for Storage Queue
- **Severity:** MEDIUM
- **File:** `ProcessingTask.cpp:51`
- **Code:** `xQueueSend(storageQueue, &r, pdMS_TO_TICKS(50));` — return value ignored
- **Issue:** Export queue failures correctly increment `g_queueDrops` (line 56), but storage queue failures are silently ignored. Storage drops are arguably *more* critical than export drops.
- **Fix:** Add `if (xQueueSend(storageQueue, ...) != pdTRUE) g_queueDrops++;`

### N9: ExportTask Silently Discards When Batch Full
- **Severity:** MEDIUM
- **File:** `ExportTask.cpp:25`
- **Code:** `if (got && batchCount < BATCH_SIZE)` — else branch missing
- **Issue:** When `got == true` but `batchCount >= BATCH_SIZE`, the reading `r` is silently discarded with no counter. The bounds check (fix M2) prevents overflow but introduces silent drop.
- **Fix:** Add drop counter in else branch.

### N10: Shutdown Race — Tasks May Enqueue After `running = false`
- **Severity:** MEDIUM
- **File:** `TaskManager.cpp:138-155`
- **Issue:** `running` set to `false` at line 139, then polls queue depths for up to 3s. But SensorTask/SlowSensorTask may still be mid-`tickFiltered()` when `running` flips — their enqueued readings arrive *after* the drain check, and are lost.
- **Mitigation:** The 3s timeout provides margin, but a proper fix would signal tasks to stop, then wait for task exit (via `eTaskGetState()`), then drain.

### N11: Ring Buffer Multi-Reader Not Guaranteed
- **Severity:** LOW
- **File:** `DataPipeline.h:35-66`
- **Issue:** SPSC (single-producer single-consumer) atomics are used, but `copyRecent()` can be called from any web handler thread. Multiple concurrent `/api/data` requests would create multiple readers — violating SPSC guarantee.
- **Mitigation:** `webDataMutex` serializes access in ProcessingTask (push side) and API handlers (read side), so the atomics are belt-and-suspenders. Not a bug with mutex present, but removing the mutex would expose the race.

## 4. NEW Findings — Sensors

### N12: `_lastReadTs = 0` Overwrites SensorManager's Timestamp (ALL Sensors)
- **Severity:** HIGH — affects 10+ sensor plugins
- **Files:** `WindSensor.cpp:84`, `SDS011Sensor.cpp:103`, `PMS5003Sensor.cpp:62`, `BME280Sensor.cpp:83,94`, `YFS201Sensor.cpp:73`, `RainSensor.cpp:60`, `WaterFlowSensor.cpp:86`, `BH1750Sensor.cpp:69`, `BME688Sensor.cpp:61`, `DS18B20Sensor.cpp:87`, `ENS160Sensor.cpp:80`, `HCSR04Sensor.cpp:58`, `SCD4xSensor.cpp:105`, `SGP30Sensor.cpp:99`, `SoilMoistureSensor.cpp:58`, `VEML6075Sensor.cpp:88`, `VEML7700Sensor.cpp:110`
- **Issue:** Every sensor plugin sets `_lastReadTs = 0` at the end of `readAll()`. SensorManager correctly calls `s->setLastReadTs(now)` at line 137 *after* `readAll()` — but the *next* call to `readAll()` immediately zeroes it again before SensorManager can set it. The `lastReadTs()` accessor always returns 0 between reads.
- **Impact:** `/api/sensors` endpoint always shows `last_read_ts: 0` for all sensors.
- **Fix:** Remove all `_lastReadTs = 0;` assignments from individual sensor plugins. Let SensorManager be the sole owner of this timestamp.

### N13: Division by Zero in YFS201Sensor / WaterFlowSensor
- **Severity:** HIGH
- **Files:** `YFS201Sensor.cpp:64-66`, `WaterFlowSensor.cpp:76-78`
- **Code:** `(float)deltaPulses / _pulsesPerLiter`
- **Issue:** If `_pulsesPerLiter` is 0 or negative (malformed config), division by zero produces NaN/Inf. `init()` validates and sets a default, but if `init()` is skipped or config is hot-reloaded, no runtime guard exists in `readAll()`.
- **Fix:** Add `if (_pulsesPerLiter <= 0) return 0;` at top of `readAll()`.

### N14: `gpio_install_isr_service(0)` Called Per-Sensor
- **Severity:** MEDIUM
- **Files:** `WindSensor.cpp:29`, `YFS201Sensor.cpp:26`, `RainSensor.cpp:25`, `WaterFlowSensor.cpp:38`
- **Issue:** Each ISR-based sensor calls `gpio_install_isr_service(0)` independently. Per ESP-IDF docs, calling this multiple times returns `ESP_ERR_INVALID_STATE` (harmless) but wastes cycles and is fragile.
- **Fix:** Call once in `TaskManager::init()` or use a static guard:
  ```cpp
  static bool isr_installed = false;
  if (!isr_installed) { gpio_install_isr_service(0); isr_installed = true; }
  ```

### N15: UART Init Never Verified (SDS011 + PMS5003)
- **Severity:** MEDIUM
- **Files:** `SDS011Sensor.cpp:18-28`, `PMS5003Sensor.cpp:15-19`
- **Issue:** Both sensors call `Serial2.begin()` and return `true` without verifying UART is functional. If Serial2 is already in use by another sensor (SDS011 + PMS5003 both use Serial2), the second `begin()` silently reconfigures the port, breaking the first sensor.
- **Impact:** Only one UART sensor can work at a time on single-UART ESP32-C3.

### N16: Float Precision Loss in RainSensor Rate Calculation
- **Severity:** MEDIUM
- **File:** `RainSensor.cpp:53-55`
- **Code:** `float rawRate = _mmPerTip / ((float)intervalUs / 3600000000.0f);`
- **Issue:** Large `uint32_t` → `float` conversion loses precision. For small intervals (~1000 us), the resulting rate (~1e9 mm/h) exceeds float mantissa (24 bits).
- **Fix:** Reorder: `_mmPerTip * 3600000000.0f / (float)intervalUs` minimizes precision loss.

### N17: WindSensor Missing `_metersPerRev > 0` Validation
- **Severity:** LOW
- **File:** `WindSensor.cpp:22-23`
- **Issue:** `_pulsesPerRev` is validated (`<= 0` → default 1.0), but `_metersPerRev` is not. If config has `"meters_per_rev": 0`, wind speed is always 0.0.

### N18: Partial-Read Contract Violation
- **Severity:** LOW
- **Files:** `YFS201Sensor.cpp:49`, `RainSensor.cpp:42`, `WaterFlowSensor.cpp:61`, `BME688Sensor.cpp:47`, `ENS160Sensor.cpp:65`, `SCD4xSensor.cpp:83`, `SGP30Sensor.cpp:88`, `SoilMoistureSensor.cpp:39`, `VEML7700Sensor.cpp:91`
- **Issue:** These sensors require `maxOut >= N` (N=2-4) and return 0 if caller provides fewer slots. ISensor interface doesn't document this contract. SensorManager always passes `maxOut=4`, so no runtime bug, but the implicit contract is fragile.

## 5. NEW Findings — Export / Storage / Web

### N19: Stale `now` in ExportManager::sendAll() Interval Check
- **Severity:** HIGH
- **File:** `ExportManager.cpp:171-189`
- **Code:**
  ```cpp
  uint32_t deadline = millis() + MAX_SENDALL_MS;  // line 171 — fresh
  uint32_t now = millis();                         // line 172 — captured once
  // ...
  if (interval > 0 && (now - _lastSentMs[i]) < interval) continue; // line 183 — stale
  _lastSentMs[i] = now;  // line 189 — records stale time
  ```
- **Issue:** `now` is captured once but used for all exporters. If early exporters take seconds (network I/O), later exporters use a stale `now` for interval checks and last-sent timestamps. This causes interval drift.
- **Fix:** Use `millis()` directly at line 183 and 189.

### N20: HttpExporter / OpenSenseMapExporter — No RAII on Heap Buffer
- **Severity:** MEDIUM
- **Files:** `HttpExporter.cpp:26-64`, `OpenSenseMapExporter.cpp:44-79`
- **Code:** `char* body = new char[bodyLen]; ... delete[] body;`
- **Issue:** If any operation between `new` and `delete[]` causes early return (unlikely in Arduino but possible with HTTP errors), `body` leaks. ESP32 doesn't throw C++ exceptions by default, so this is low-risk but poor practice.
- **Fix:** Use `std::unique_ptr<char[]>` or cleanup-on-error pattern.

### N21: Stale Pointer Risk in API Parameter Capture
- **Severity:** MEDIUM
- **File:** `ApiHandlers.cpp:86-87`
- **Code:**
  ```cpp
  const char* aggParamStr  = req->hasParam("agg") ? req->getParam("agg")->value().c_str() : "5m";
  const char* modeParamStr = req->hasParam("mode") ? req->getParam("mode")->value().c_str() : "lttb";
  ```
- **Issue:** `c_str()` returns pointer into `String` object owned by `AsyncWebServerRequest`. If the request object's parameter storage is recycled during async response writing, the pointer dangles.
- **Fix:** Copy into local `char[]` or `String` variables.

### N22: JsonLogger Line Buffer Silently Drops Long Lines
- **Severity:** MEDIUM
- **File:** `JsonLogger.cpp:72-81`
- **Code:** `char line[128]; int n = r.toJsonLine(line, sizeof(line)); if (n <= 0 || n >= (int)sizeof(line)) return;`
- **Issue:** If a `SensorReading` with long string fields exceeds 127 bytes, the line is silently dropped. No warning logged. With 16-char `sensorId` + 16-char `sensorType` + 16-char `metric` + 8-char `unit` + timestamp + value, a full line can reach ~120 bytes — leaving only 7 bytes margin.
- **Fix:** Increase to 160 bytes and/or log a warning on drop.

### N23: HybridStorage `mirrorWrite()` Ignores Write Failures
- **Severity:** LOW
- **File:** `HybridStorage.cpp:62-75`
- **Issue:** `f.write(data, len)` return value not checked. If SD card is full or LittleFS partition is exhausted, partial writes occur silently.
- **Fix:** Check `f.write()` return value, log warning if `!= len`.

### N24: Timezone Ambiguity in `_fileDateInRange()`
- **Severity:** LOW
- **File:** `JsonLogger.cpp:133-148`
- **Issue:** Uses `mktime()` which interprets dates in local timezone. The ±1 day safety margin compensates but wastes I/O by reading extra files.
- **Mitigation:** Already mitigated by safety margin. Document as known limitation.

## 6. NEW Findings — Managers & Utilities

### N25: ConfigManager — Unsafe `strcpy()` Without Bounds Checking
- **Severity:** HIGH
- **File:** `ConfigManager.cpp` — lines 39-40, 43-45, 77, 80-94, 195-210, 264-266, 291
- **Issue:** Multiple `strcpy()` calls on user-configurable strings without size limits. If config JSON contains strings longer than the destination buffer, stack/heap corruption occurs.
- **Fix:** Replace all `strcpy()` with `strncpy(dst, src, sizeof(dst)-1); dst[sizeof(dst)-1] = '\0';`

### N26: ConfigManager — No Future-Version Guard in Migration
- **Severity:** MEDIUM
- **File:** `ConfigManager.cpp:290-310`
- **Issue:** `migrateConfig()` handles `fromVersion < N` cases but doesn't check `fromVersion > CONFIG_VERSION`. Loading config from newer firmware silently applies old-version migrations, potentially corrupting fields.
- **Fix:** Add `if (fromVersion > CONFIG_VERSION) { applyDefaults(); return; }`

### N27: ConfigManager — `applyDefaults()` Not Thread-Safe
- **Severity:** MEDIUM
- **File:** `ConfigManager.cpp:11-95`
- **Issue:** Modifies global `config` struct without any mutex. If called during config hot-reload while other tasks read `config`, data races occur.
- **Fix:** Wrap in `configMutex` or ensure it's only called during init.

### N28: WiFiManager — Busy-Loop Without Delay
- **Severity:** MEDIUM
- **File:** `WiFiManager.cpp:73-78`
- **Code:** `while (WiFi.status() != WL_CONNECTED && ...) { yield(); esp_task_wdt_reset(); }`
- **Issue:** Tight loop with only `yield()` and WDT reset. No `delay()` means CPU runs at 100% during WiFi connect (~5-15s), wasting power and preventing lower-priority tasks from running.
- **Fix:** Add `delay(100)` inside the loop.

### N29: StorageManager — `activeFS` Null Dereference Risk
- **Severity:** MEDIUM
- **File:** `StorageManager.cpp:103`
- **Code:** `File dir = activeFS->open(path);`
- **Issue:** `getCurrentViewFS()` can return `nullptr`. Callers may not null-check before dereferencing.
- **Fix:** Add null guard at each call site.

### N30: DataLogger — `logBuffer` Discards Without Flushing
- **Severity:** MEDIUM
- **File:** `DataLogger.cpp:171-176`
- **Issue:** When `logBufferCount >= LOG_BATCH_SIZE`, oldest entry is shifted out without first attempting to flush to filesystem. Data is silently lost.
- **Fix:** Call `flushLogBufferToFS()` before discarding.

### N31: HardwareManager — ISR Debounce Variables Not Atomic
- **Severity:** MEDIUM
- **File:** `HardwareManager.cpp:10-18`
- **Code:** ISR reads and writes `lastFFInterrupt` (read-modify-write) without atomics.
- **Issue:** On ESP32-C3 (single core), ISRs can't truly interleave, so this is safe. On dual-core ESP32, this would be a race condition. Code is not portable.
- **Mitigation:** Safe on target hardware (XIAO ESP32-C3 is unicore).

### N32: RtcManager — Missing Null After Delete
- **Severity:** LOW
- **File:** `RtcManager.cpp:26-27`
- **Code:** `if (Rtc) delete Rtc;` — doesn't set `Rtc = nullptr` after delete.
- **Issue:** Double-free risk if `initRtc()` is called twice without reinit.
- **Fix:** Add `Rtc = nullptr;` after each `delete`.

### N33: WiFiManager — DST Offset Hardcoded to 0
- **Severity:** LOW
- **File:** `WiFiManager.cpp:117`
- **Code:** `configTime(config.network.timezone * 3600, 0, config.network.ntpServer);`
- **Issue:** DST offset is always 0. Devices in DST regions will show incorrect local time.
- **Fix:** Add `dst_offset_s` field to network config.
